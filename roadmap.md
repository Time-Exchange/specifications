# Time-Exchange Development Roadmap

This roadmap outlines the transition from a philosophical concept to a global, decentralized time-exchange infrastructure.

## Phase 1: Foundation & Specification (Current)
- [x] Define the Manifesto (1h = 1h)
- [x] Establish Architecture Patterns (Zero-cost, P2P)
- [x] Design the Protocol (Data structures & Handshakes)
- [ ] Community setup and initial contributor guidelines.

## Phase 2: Core Engine Development (The Heart)
- [ ] Initialize `core-engine` repository.
- [ ] Implement P2P networking using `libp2p`.
- [ ] Create the Cryptographic Identity Module (Ed25519).
- [ ] Develop the Local Ledger (SQLite/NoSQL) for offline-first capabilities.

## Phase 3: Trust & Reputation System
- [ ] Define the "Proof of Service" verification logic.
- [ ] Implement the reputation scoring algorithm (anti-fraud).
- [ ] Beta testing of the matching engine in a CLI environment.

## Phase 4: User Experience (The Interface)
- [ ] Initialize `mobile-interface` (Flutter/React Native).
- [ ] Wallet integration (Storing and transferring TUs).
- [ ] QR-code based peer discovery and transaction signing.

## Phase 5: Global Scaling & Stabilization
- [ ] Stress-testing the Gossip protocol for 1M+ nodes.
- [ ] Localization for major global languages.
- [ ] Launching the decentralized "Visualizer" dashboard.
